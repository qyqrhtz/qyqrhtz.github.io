关于SQL

索引

1.按存储形式分类:
聚簇索引,一般是id,只有一个
二级索引,多个

2.索引结构:
B+Tree R tree hash索引 fulltext(全文本?)索引

为什么 InnoDB 存储引擎选择使用 B+Tree 索引结构？
答:平衡,节约空间,只有叶子结点存数据,支持范围匹配及排序操作,查询效率高
InnoDB 主键索引的 B+Tree 高度为多少？
答:答：假设一行数据大小为1k，一页中可以存储16行这样的数据。InnoDB 的指针占用6个字节的空间，主键假设为bigint，占用字节数为8.
可得公式：n * 8 + (n + 1) * 6 = 16 * 1024，其中 8 表示 bigint 占用的字节数，n 表示当前节点存储的key的数量，(n + 1) 表示指针数量（比key多一个）。算出n约为1170。

索引类型:

主键索引:只有一个,一般默认创建primary
唯一索引:有多个,避免列数据重复unique
常规索引:多个,快速定位特定数据
全文索引:多个,全文范围索引,查找关键字不比较值fulltext

语法

创建索引：create 类型index 索引名字 on 表(字段)
查看:show index from 表
删除:drop index 索引名 on 表
explain查看执行计划
查询快慢 null system constant eq_ref ref

最左前缀法则,可以用<=不能只用<

语法练习:

继承:✅ 本质：子类自动拥有父类的属性和方法

 ⚠️ 注意：

 Java 单继承：子类只能继承一个父类

子类可以访问父类的非 private 成员，但 private 的需要通过方法访问

多态:✅ 本质：同一个父类引用，不同子类对象表现出不同行为

 ⚠️ 纠正：

“子类用不同于父类的方法” → 更准确说法是 子类重写父类方法，父类引用指向子类对象时，执行子类的方法

接口:可以定义多个方法,只声明方法,不关注实现(那什么关注实现)

封装:子类和外部类不可以直接调用父类的属性,只能通过特定方法访问

oop;面相对象编程

重载:重写父类方法

构造函数(构造器):初始化当前类对象,如果是子类,默认先调用父类构造器

this关键字区分作用域
final关键字定义常量

关于sql

学了事务
原子性就是事务不可分割,
一致性就是事务完成了数据要一致,
隔离性就是两个事务互不干扰,
持久性就是事务commit了就永久有效…

还有几个并发事务的问题:
脏读就是事务1读取了事务2修改过的数据
read commited可以解决
不可重复读就是事务1读取id1,如然后事务2修改id1,事务1再读id1结果和第一次不一样
repeatable read 可以解决
幻读比较抽象,事务1读取id1没读到,事务2插入id1,事务1想插入发现已经有id1了
serializable可以解决(记这些有啥用…)
